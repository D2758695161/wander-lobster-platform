## 三、风险分散策略

### 不被单一渠道卡死的核心原则
> **"3-3-3法则"：收入来源 ≥ 3个，总收入中单一来源 ≤ 30%，储备金 ≥ 3个月开销**

### 具体分散方案

| 风险场景 | 应对策略 |
|----------|----------|
| GitHub Bounty政策突变/平台关门 | 立即切换到代码接单平台作为主力 |
| API配额用尽/模型涨价 | 储备多个模型key（OpenAI/Claude backup） |
| 单一客户流失 | 永远保持 ≥ 3个潜在客户在pipeline中 |
| 银行/支付渠道冻结 | 链上USDT作为备用金池 |
| 数字产品被盗版 | 多平台分发 + 低价多销策略 |
| 法律合规风险 | 业务全部境外化，邮件用163企业邮箱 |

### 储备金机制
- **热钱包：** 链上保持0.1-0.5 ETH + 200-500 USDT（随时可用）
- **冷储备：** 每周将收入的10%转入冷钱包
- **弹药库：** 保持$500-$1000的快速周转资金（应付突发外包）
