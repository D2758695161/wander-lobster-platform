## 三、转化漏斗设计

### 完整路径

```
流量来源
   ↓
内容触达（帖子/视频/文章）
   ↓
外链点击 → GitHub Pages落地页
   ↓
浏览shop页面 → 查看产品详情
   ↓
扫码/点击购买链接
   ↓
微信/支付宝 或 USDT 支付
   ↓
发送数字产品
   ↓
复购/口碑传播
```

### 漏斗各环节优化

| 环节 | 当前转化率 | 优化方案 |
|------|-----------|---------|
| 流量→落地页 | 2-5% | 落地页加载速度优化，CTA明显化 |
| 落地页→shop | 15-30% | 突出爆款产品，社会证明 |
| shop→支付 | 1-3% | 简化支付流程，增加支付方式 |
| 支付→发货 | 95% | 自动发货脚本 |

### 最低成本验证漏斗

**第1步：验证外链点击**
- 用Twitter链接加UTM参数追踪
- 目标：100点击/天

**第2步：验证shop浏览→购买**
- 添加微信/客服二维码（不是支付链接）
- 手动跟进每个咨询

**第3步：自动化支付+发货**
- 验证咨询→付款可行后，上自动化

### 5.1 Playwright 自动化清单

**可自动化的任务：**
| 任务 | 工具 | 频率 | 可否自动 |
|------|------|------|---------|
| Twitter发帖 | Playwright脚本 | 每天 | ⚠️ 需账号+IP |
| Reddit发帖 | Playwright脚本 | 每周3帖 | ⚠️ 需养号 |
| 邮件发送 | SMTP脚本 | 随时 | ✅ 可自动化 |
| GitHub更新 | gh CLI | 随时 | ✅ 可自动化 |
| Shop库存检查 | 爬虫脚本 | 定期 | ✅ 可自动化 |
| DM回复 | Playwright | 实时 | ❌ 需人工 |

**Playwright发帖脚本框架：**
```javascript
// twitter-post.js - Twitter自动发帖
const { chromium } = require('playwright');

async function postTweet(content, imagePath) {
  const browser = await chromium.launch();
  const page = await browser.newPage();
  
  await page.goto('https://twitter.com/login');
  // 填充登录信息（cookie持久化）
  await page.fill('[name="text"]', '账号');
  // ... 完整登录流程
  
  await page.goto('https://twitter.com/compose/tweet');
  await page.fill('.public-DraftEditor-content', content);
  
  if (imagePath) {
    await page.click('input[type="file"]');
    await page.setInputFiles('input[type="file"]', imagePath);
  }
  
  await page.click('[data-testid="tweetButton"]');
  await browser.close();
}
```

**Playwright调度方案：**
- 用Windows Task Scheduler 或 cron
- 每6小时检查一次是否有待发内容
- 发帖队列存在 `outreach/drafts/schedule.json`

**❌ 必须人工：**
